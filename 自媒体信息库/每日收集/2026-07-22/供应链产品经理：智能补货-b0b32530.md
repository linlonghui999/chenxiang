# 供应链产品经理：智能补货

## 信息摘要

仓库补货是供应链中最考验产品经理能力的环节之一。本文深入解析如何通过ABC价值分层与XYZ波动分类，构建九宫格联合补货策略，并详细拆解补货点、触发条件及补货量的计算公式，助你实现不缺货、不积压的智能补货体系。 供应链分为采购，物流，仓储。其中仓储是整个供应链环节中最复杂的一环，作为产品经理要了解出库，入库，库内三个基本环节，每个环节中又包含多个子流程，比如出库，包含出库订单，拆单，组波，任务分配，拣货等一系列流程，内个流程都对应不同的模块，所以，作为产品经理能把仓库业务搞透，是个需要时间积累经验的事。 在仓库业务中，能不能保证货够买，还不压库存，是个富有挑战性的工作，作为产品经理，在设计系统时，怎么做补货，就成了看产品经理能力的关键节点。 下面我分享我做仓库补货时的想法，主要从ABC价值分层，XYZ波动分层，来采用不用的补货策略。 目标与任务 总体目标 ： 建立数据驱动的智能补货体系，以 ABC分级 分配资源优先级、以 XYZ波动 差异化设计安全库存、还有 服务系数 计算，实现不缺货、不积压、少搬运。 具体目标 ： 缺货停线≤1次/月、补货频次降20%-30%、退货时效36h、系统年化成本降40%+、拣货效率提升40%+。 任务一：ABC×XYZ联合分级 分类逻辑： ABC决定”谁更重要”（销量贡献度），XYZ决定”需求稳不稳定”（波动系数CV=标准差/均值） ABC分层： 拆零拣货贡献度，是 拆零拣货业务在仓库整体出库、成本、工时、营收中所占的比重 ，用来量化碎片化小件订单对仓库运营的影响，区分整件 / 整托拣货、拆零拣货两大业务板块的价值与成本消耗。 XYZ波动分类： X类（CV < 0.5，高可预测）、Y类（0.5≤CV < 1.0，中可预测）、Z类（CV≥1.0，低可预测）。 静态波动： 过去6个月日销量标准差/均值，每月更新，定基础安全库存系数 动态波动： 近7天 vs 前30天均值的偏离程度，每日实时，旺季临时调整安全系数 ABC×XYZ九宫格联合策略： 九宫格详细参数： 任务二：补货触发条件与公式 1. 补货点也就是再订货点（ROP）计算 补货点（ROP）= 提前期需求量 + 安全库存 提前期需求 = 日均预测销量 × 补货提前期（主仓→拣选仓搬运+上架，0.5-1天） 安全库存 = 服务系数 × 日销量标准差 × √提前期 × 动态波动系数 服务系数（正态分布分位数）： 动态波动系数： 平峰期为1.0，旺季/大促期间上调（基础1.5倍，A-Z类最高2.5倍）。 各矩阵 ROP 计算示例：（举例子） 2. 补货触发条件 基础触发： 拣货位实时库存 ≤ ROP时触发补货 动态波动触发（实时调整动态波动系数）： 旺季系数联动： 平峰：动态波动系数 = 1.0 旺季：动态波动系数 = 基础值 × 1.5 例：A-Y旺季动态波动系数 = 1.0×1.5 = 1.5，A-Z旺季 = 1.0×1.5 = 1.5（在此基础上A-Z可进一步上调至2.5） 退货回流触发： 退货质检完成+结果为”可售”→自动生成退货回流补货任务（黄色优先级），避免假性缺货。 波次协同触发： 波次释放前30分钟→预判波次涉及SKU→检查拣货位库存是否足够→不够则提前触发。 3. 建议补货量计算 建议补货量 = MAX(补货点（ROP） – 当前拣选仓可用库存 – 在途补货量 + 已锁未拣量, 最小补货单位) 最小补货单位： 按箱规/托盘规，童装散货通常为 1箱（约50件） 。 三层约束校验（取最小值兜底）： 实际补货量 = MIN(建议补货量,拣货位剩余容量 × 80%（留20%缓冲）,存储区可用库存) 完整计算示例（A-Y类 SKU ）： 基础参数： 日均预测销量：500件 | 提前期：0.5天 服务系数：1.65（A类95%）| 日销量标准差：120件 动态波动系数：1.0（平峰） ROP = 500×0.5 + 1.65×120×√0.5×1.0 = 250 + 140 = 390件 当前状态： 拣货位可用库存：150件 | 在途补货量：0件 已锁未拣量：20件 | 拣货位最大容量：500件 存储区可用：200件 | 最小补货单位：50件 建议补货量 = MAX(390 – 150 – 0 + 20, 50) = MAX(260, 50) = 260件 三层约束校验： 约束一：拣货位剩余容量×80% = (500-150)×80% = 280件 约束二：建议补货量 = 260件 约束三：存储区可用库存 = 200件 实际补货量 = MIN(280, 260, 200) = 200件 → 补200件，受存储区库存限制 → 系统标记”存储区库存不足”，推送采购预警 本文由 @会员 原创发布于人人都是产品经理。未经作者许可，禁止转载
