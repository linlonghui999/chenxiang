# 属性规范

> 属性负责筛选和维护，链接负责表达关系，文件夹只负责生命周期和大范围归属。  
> 本库的属性设计优先服务“资料 → 观点 → 输出”的转化，而不是追求元数据完整。

## 核心属性

- `title`：笔记标题。
- `type`：笔记类型。
- `domain`：领域。
- `category`：领域内主题。
- `status`：当前处理状态。
- `maturity`：成熟度。
- `value`：对当前目标的价值。
- `use_case`：这条笔记最适合服务什么输出。
- `audience`：主要面向谁。
- `claim_strength`：笔记中主张的可靠程度。
- `next_action`：下一步动作。
- `up`：上级领域或系统入口。
- `map`：直接所属的分类 MOC。
- `source` 或 `sources`：资料出处和证据来源。

## 推荐枚举

### `type`

- `source`：原始资料、摘录、网页、文件、访谈、视频转写。
- `evergreen`：提炼卡片，即可独立引用的观点。
- `moc`：主题地图。
- `project`：内容项目、产品项目、课程项目。
- `review`：复盘、体检、审查。
- `daily`：每日记录。
- `dashboard`：工作台。
- `system`：规则、规范、说明。
- `template`：模板。
- `archive`：历史资料或测试文件。

### `domain`

- `沉香`
- `修行`
- `自媒体`
- `跨领域`
- `待判断`

### `status`

- `待整理`：刚进入系统，还没判断价值。
- `待阅读`：已保存，等待阅读标注。
- `待提炼`：值得保留，需要提炼观点。
- `生长中`：正在形成观点或结构。
- `构思中`：项目尚未发布。
- `进行中`：项目或复盘正在推进。
- `已发布`：内容已经发布。
- `已归档`：低频、过时、测试或暂不处理。
- `持续更新`：入口、MOC、系统文件。
- `稳定`：规则基本稳定，只在必要时调整。

### `maturity`

- `seed`：刚提出，证据、边界、连接都不足。
- `growing`：已有来源支持，开始出现边界、反例或应用。
- `evergreen`：多次使用后仍稳定，能支撑长期输出。

### `value`

- `高`：能直接支撑观点、选题、脚本、产品或关键判断。
- `中`：有参考价值，但需要更多证据或转化。
- `低`：仅作背景、存档或暂不处理。
- `待判断`：尚未阅读或价值不明。

### `use_case`

- `选题`：能生成标题、钩子或选题方向。
- `脚本`：能直接进入短视频、图文、直播结构。
- `证据`：提供事实、数据、案例、出处。
- `案例`：可作为故事、人物、行业事件。
- `产品`：可用于课程、咨询、销售、话术。
- `方法`：可转化为步骤、练习、流程。
- `反例`：用于边界、风险、误区、辟谣。
- `待判断`：暂未确定用途。

### `audience`

- `新手`
- `进阶`
- `客户`
- `同行`
- `泛流量`
- `自己`
- `待判断`

### `claim_strength`

- `事实`：可查证事实、政策、数据、明确出处。
- `研究`：来自论文、专著、专业资料或系统研究。
- `经验`：来自实践者、个人观察、案例复盘。
- `推测`：有逻辑但证据不足。
- `营销说法`：可能有商业包装，需要谨慎使用。
- `需验证`：暂不可直接作为强主张。

### `next_action`

- `阅读`
- `补来源`
- `提炼卡片`
- `连接MOC`
- `做选题`
- `写脚本`
- `归档`
- `暂缓`

## 使用原则

1. **少填但要关键**：`value`、`use_case`、`claim_strength` 比复杂分类更重要。
2. **资料不等于观点**：`source` 只保存证据，自己的判断进入 `evergreen`。
3. **不确定要显式标出**：不要把营销说法、经验判断、研究结论混在一起。
4. **文件夹不承载全部意义**：同一资料可属于沉香，也可服务自媒体，用属性和链接表达多重用途。
5. **每次只推进一个动作**：看完一条资料后，至少明确 `next_action`。
