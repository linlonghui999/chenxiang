# 我把 Codex 主题做成了一个真正的管理器

## 信息摘要

Codex Theme Creator 从命令行工具升级为桌面 App，让 Apple 芯片 Mac 用户一键安装、预览和切换主题。本文详解其主题库管理、实时预览、菜单栏切换及 AI 创作新主题等核心功能，展示如何降低主题定制门槛，提升 Codex 使用体验。 前几天，我开源了 Codex Theme Creator。最初的版本已经能让 Codex 根据一句描述或一张参考图创作主题，但对普通用户来说，安装、保存和切换仍有些麻烦。 这两天，我把它做成了一个真正可以安装的桌面 App。 现在，Apple 芯片 Mac 用户下载 DMG、把 App 拖进“应用程序”，就可以直接使用主题库，不需要先克隆仓库，也不用照着教程敲命令。 现在能做什么 打开 App，首页就是本机主题库。公开预设和自己创作的主题都会显示在这里。 选中一套主题后，可以直接预览，并调整强调色、背景模糊、背景压暗、首页卡片透明度和任务面板透明度。满意后点击“保存并启用”，主题就会写入 Codex。 目前已经完成的功能包括： 主题库管理和实时预览； 一键保存、启用和恢复默认外观； 强调色、模糊与界面透明度微调； 导入、导出 .ctheme 主题包； 菜单栏快速切换常用主题； 用一句描述或参考图，让 Codex 创作新主题。 我很喜欢菜单栏这个入口。平时不用一直开着管理器，需要换状态时点一下就行：写代码可以用赛车主题，晚上整理想法可以切到星空，想安静写东西时换成雨林。 新主题还是交给 Codex 来做 第一次打开 App 时，它会自动安装配套的 Creator Skill。之后点击“让 Codex 创作新主题”，把自己的想法发给 Codex 即可。 比如： 做一套 GT 赛车主题。雨后赛道、深色、红色强调色，文字必须清楚。 也可以直接附上一张参考图。 Codex 会处理背景、配色、侧栏、选中状态、输入框和任务页面，并把完成的主题写入本机主题库。回到管理器刷新一下，就能继续预览、微调和启用。 主题文件和图片都保存在本机，也可以导出后发给别人。这个项目不会修改 Codex 的 app.asar 或应用签名；遇到 Codex 更新导致的兼容问题，也可以从菜单栏恢复默认外观。 目前的版本 macOS 测试体验版已经上线，目前支持 Apple 芯片 Mac，需要预先安装 Codex Desktop，建议把 Codex 设置为深色外观。 Windows 版本正在适配。界面和主题包结构已经预留，我会等真实的主题应用和切换链路跑通后再提供安装包。 后续我也计划成熟主题分享机制 这是一个非官方开源项目，与 OpenAI 没有隶属关系。如果你也在用 Codex Desktop，欢迎下载试用，也欢迎在 GitHub 提 issue，把使用中遇到的问题告诉我。 项目主页： https://swording-k.github.io/codex-theme-creator/ GitHub： https://github.com/swording-k/codex-theme-creator 本文由 @宝剑 原创发布于人人都是产品经理。未经作者许可，禁止转载 题图来自Unsplash，基于CC0协议
