# 知识库健康检查

> 每周检查的目的不是追求全库整齐，而是保证“资料输入 → 观点提炼 → 内容输出 → 反馈修正”的链路持续运转。

## 每周检查

### 输入层

- [ ] 收件箱是否有超过 7 天未处理的资料？
- [ ] 是否存在只收藏、未阅读、未解释的资料？
- [ ] 本周新增资料是否标记了 `value`、`use_case`、`claim_strength`？
- [ ] 是否有明显重复资料需要列入审查清单？

### 提炼层

- [ ] 本周是否至少提炼 3 张观点卡片？
- [ ] 每张新卡片是否有来源、边界、反例或横向关系？
- [ ] 是否有高价值资料长期停留在 `待阅读` 或 `待提炼`？

### 主题层

- [ ] 新笔记是否都连接了领域 MOC 和分类 MOC？
- [ ] 哪个 MOC 已经退化成目录，需要补“可靠结论/争议/空白”？
- [ ] 哪个主题已经足够形成一篇内容或一期视频？
- [ ] 哪两张笔记之间出现了支持、冲突或新的组合？

### 输出层

- [ ] 本周是否从知识库生成至少 3 个选题？
- [ ] 是否有内容项目连接了至少 3 张知识卡片？
- [ ] 已发布内容带来了哪些反馈，需要修正哪张卡片？

## 健康标准

- 每条来源资料至少有 2 个出链：领域入口和分类地图。
- 每张提炼卡片至少有 1 个来源、1 个 MOC 和 1 个横向关系。
- 每个活跃主题 MOC 都包含：核心问题、来源、提炼观点、可靠结论、争议、知识空白、可输出选题。
- 每个内容项目至少连接 3 张卡片：背景、核心观点、案例/反例/方法。
- 收件箱保持可清空，不把“待处理”永久当作分类。
- 高价值资料不能长期停留在 `待阅读`，必须进入 `待提炼`、`做选题` 或 `归档`。

## 每周最小动作

- [ ] 处理 3 条待阅读资料。
- [ ] 提炼 3 张观点卡片。
- [ ] 更新 1 个 MOC。
- [ ] 生成 3 个内容选题。
- [ ] 把 1 条发布反馈写回相关卡片。

## 月度对抗性审查

- 哪些分类只是因为“看起来应该有”，但没有服务输出？
- 哪些资料反复出现，却没有形成自己的判断？
- 哪些观点证据不足，却已经被当作强结论？
- 哪些主题最能形成“沉香 × 修行 × 自媒体”的独特定位？
- 哪些低价值来源应该停止收集？
