# 知识库体检报告

> 体检时间：2026-07-22  
> 范围：仅扫描 Markdown 笔记，不移动、不删除、不批量改动原始资料。  
> 核心判断：当前知识库的主要瓶颈不是“没有架构”，而是“资料尚未被稳定压缩成观点、判断和内容输出”。

## 1. 第一性原理

本知识库的目的不是保存更多资料，而是服务一条明确链路：

```text
资料输入 → 可信度判断 → 观点提炼 → 主题判断 → 内容输出 → 反馈修正
```

因此，健康标准不是笔记数量，而是：

- 能不能快速找到能用的沉香、修行、自媒体素材？
- 能不能区分事实、经验、推测和营销说法？
- 能不能从资料中提炼出自己的观点？
- 能不能持续生成短视频、图文、直播、课程或产品话术？
- 能不能把发布反馈写回知识卡片，让观点越来越准？

## 4. 对抗性审查

### 4.1 最大风险：资料坟场

`source` 占比很高，但提炼卡片很少。若继续只收藏资料，知识库会越来越像资料仓库，而不是能支撑创作和判断的系统。

**修正方向**：每周固定从高价值资料中提炼观点卡片，而不是追求全量整理。

### 4.2 MOC 容易退化成目录

目前 MOC 数量足够，但大量笔记没有实际语义连接。MOC 如果只是链接列表，无法帮助判断“哪些观点重要、哪些证据冲突、哪些内容可输出”。

**修正方向**：MOC 必须回答核心问题、可靠结论、争议空白、可输出观点。

### 4.3 重复资料会消耗注意力

存在同名资料同时出现在沉香库和自媒体库的情况。重复不一定要立刻删除，但需要标记主副本，否则后续提炼会重复劳动。

**修正方向**：先建立重复清单，审查后再决定保留、合并或归档。

### 4.4 没有输出约束，整理会变成学术洁癖

如果只按领域整理，会越分越细；如果以输出倒推，资料会自然分出优先级。

**修正方向**：每条高价值资料都要判断它能服务哪个 `use_case`：选题、脚本、证据、案例、产品、待判断。

## 5. 本轮迭代目标

本轮只升级系统规则和模板，不处理原始资料：

- 建立体检报告，暴露当前系统问题。
- 升级属性规范，让资料进入时就带有输出意图。
- 升级链接规范，要求链接写出关系。
- 升级资料、卡片、项目模板，让提炼直接导向内容生产。
- 升级健康检查，把“是否产生观点和输出”作为核心指标。

## 6. 下一步建议

### 建议先做 10 篇样板提炼

优先选择：

1. 沉香 × 自媒体交叉资料。
2. 沉香真伪、价格、文化、药用资料。
3. 修行中能转化为大众表达的资料。
4. 自媒体 AI 工具和短视频方法论资料。

目标不是全库整理，而是证明这条链路可行：

```text
1 篇资料 → 1 个判断 → 1 张观点卡片 → 1 个选题
```

## 7. 每周最小维护动作

- 处理 3 条待阅读资料。
- 提炼 3 张观点卡片。
- 更新 1 个 MOC 的“可靠结论/争议/可输出观点”。
- 生成 3 个内容选题。
- 把已发布内容的反馈写回相关卡片。
